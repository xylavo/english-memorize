# 스토리텔링 / 연쇄법 (Story Method / Narrative Chaining)

## 개요

여러 단어(또는 한 단어의 여러 의미·연어)를 하나의 짧은 이야기 줄거리에 순서대로 엮어 넣는다. 이야기는 인과관계와 감정이 있어 단순 나열보다 훨씬 잘 기억된다. 추상어나 숙어처럼 직접적인 이미지화가 어려운 항목에 특히 유용하다.

## 적합한 입력

- 추상 명사, 감정/태도를 나타내는 단어 (예: reluctant, ambitious, skeptical)
- 숙어, 구동사 (phrasal verbs, idioms)
- 함께 외우면 좋은 단어 묶음 (3~6개)

## 절차

1. **등장인물/배경 설정**: 간단한 인물(또는 사용자 자신)과 장소를 정한다.
2. **순서 배치**: 입력된 단어(들)를 이야기의 자연스러운 흐름(기-승-전-결 또는 시간 순서) 속에 하나씩 등장시킨다.
3. **의미 강조**: 각 단어가 등장하는 문장에서 그 단어의 뜻이 문맥상 분명히 드러나도록 쓴다. 단어는 굵게 표시.
4. **짧게 유지**: 전체 이야기는 3~6문장 이내로 압축한다.

## 출력 형식

```
**스토리텔링**
<3~6문장의 짧은 이야기. 단어는 **굵게** 표시>

각 단어 의미:
- **<단어1>**: <뜻>
- **<단어2>**: <뜻>
```

## 예시

단어 묶음: reluctant(내키지 않는), postpone(연기하다), eventually(결국)

```
**스토리텔링**
민수는 약속에 가는 게 **reluctant**(내키지 않는) 했다. 비가 너무 많이 와서 그는 약속을 **postpone**(연기하다)하기로 했다. 친구는 처음엔 화났지만, **eventually**(결국) 이해해주었다.

각 단어 의미:
- **reluctant**: 내키지 않는, 마지못한
- **postpone**: 연기하다, 미루다
- **eventually**: 결국, 마침내
```

## 주의

- 너무 많은 단어(7개 이상)를 한 이야기에 넣으면 오히려 혼란스러워진다 — 그럴 땐 mnemonic-memory-palace로 전환을 고려한다.
- 이야기는 황당하거나 과장될수록 기억에 더 오래 남는다.
