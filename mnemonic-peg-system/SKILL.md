# 페그법 (Peg System)

## 개요

메모리 팰리스가 공간(장소) 경로를 외워 거기에 단어를 거는 방식이라면, 페그법은 미리 외워둔 고정된 "걸이(peg)" 이미지 1~10개에 단어를 거는 방식이다. 매번 새 경로를 설계할 필요 없이 항상 같은 페그 세트를 재사용할 수 있어, 10개 이하의 짧은 순서 목록을 빠르게 외울 때 메모리 팰리스보다 가볍다.

## 적합한 입력

- 10개 이하의 순서가 있는 단어/항목 목록
- 메모리 팰리스용 경로를 새로 만들기 번거로운 상황
- 발표 핵심어, 순서가 중요한 문법 항목(예: 조동사 변화 순서) 등

부적합: 단일 단어 하나만 외울 때, 또는 11개를 초과하는 긴 목록(이 경우 메모리 팰리스나 페그 세트 2벌 분할을 고려).

## 표준 숫자-모양 페그 세트 (1~10)

| 숫자 | 페그 이미지 | 모양 연상 |
|---|---|---|
| 1 | 막대/연필 | 숫자 1처럼 길고 곧음 |
| 2 | 백조 | 숫자 2처럼 목이 굽음 |
| 3 | 이어폰/귀 | 숫자 3처럼 굴곡짐 |
| 4 | 의자/돛 | 숫자 4의 꺾인 다리 모양 |
| 5 | 낚시 바늘 | 숫자 5의 갈고리 모양 |
| 6 | 달팽이/파이프 | 숫자 6의 동그란 꼬리 |
| 7 | 절벽/도끼 | 숫자 7의 꺾인 모서리 |
| 8 | 눈사람 | 숫자 8의 두 덩어리 |
| 9 | 풍선(끈 달림) | 숫자 9의 둥근 머리+꼬리 |
| 10 | 야구배트+공 | 1(배트)과 0(공) |

(사용자가 이미 익숙한 다른 페그 세트가 있다면 그것을 우선 사용한다.)

## 절차

1. 입력된 항목 목록을 순서대로 나열한다 (최대 10개, 넘으면 분할 안내).
2. 각 항목을 순서에 맞는 페그 이미지와 결합한 과장된 장면을 만든다.
3. 페그 이미지가 주체가 되어 항목의 의미를 "행동"하거나 "변형"되는 형태로 만들면 더 잘 붙는다.
4. 전체 페그-항목 매핑을 번호 순서대로 정리해 제시한다.

## 출력 형식

```
**페그법 (1~10 고정 페그)**
1. 막대 → <항목1> : <장면 묘사>
2. 백조 → <항목2> : <장면 묘사>
...
```

## 예시

목록: economy(절약하다), reluctant(꺼리는), benevolent(자비로운)

```
**페그법 (1~10 고정 페그)**
1. 막대(연필) → economy : 가늘고 길게 깎은 연필을 다 닳을 때까지 아껴 쓴다 — economy(절약)
2. 백조 → reluctant : 백조가 차가운 물에 들어가길 꺼리며(reluctant) 목을 빼고 머뭇거린다
3. 이어폰 → benevolent : 이어폰에서 자비로운(benevolent) 목소리의 기부 광고가 흘러나온다
```

## 주의

- 페그 세트는 한 번 정하면 계속 같은 것을 재사용해야 효과가 쌓인다 — 매번 다른 이미지로 바꾸지 않는다.
- 항목이 10개를 넘으면 두 번째 페그 세트(11~20 등)를 새로 정의하거나 메모리 팰리스로 전환을 고려한다.
